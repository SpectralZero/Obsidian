# How to Use This Book

This vault is designed to produce real capability, not passive knowledge.

## The Learning Contract

If you only read, you will forget.
If you read + do labs + write your own notes, you will build skill.

Rule of thumb:

- 30% reading
- 60% hands-on
- 10% writing (your own summaries + checklists + mistakes log)

## The Chapter Pattern

Most chapters follow this structure:

- What you must be able to explain (interview-level)
- Deep explanation (systems view)
- Attacker view (how it breaks)
- Defender view (how it gets detected / prevented)
- Walkthrough labs (commands + expected outcomes)
- Exercises (no hints)
- Self-test checklist

## Your Personal Notebook (Create These Files)

- My_Mistakes_Log.md (every failure becomes a lesson)
- My_Command_Cheatsheet.md (your own commands, not copy/paste)
- My_Detection_Notes.md (what logs prove what happened)

## How to Practice Like a Professional

- build a mental model first (how the system should work)
- observe reality (packets, logs, processes)
- break it in the lab (controlled)
- detect it (find evidence)
- report it (clear narrative, impact, proof)
